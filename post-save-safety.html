Build 42 Save Safety: How to Update Without Losing Your Progress

New May 9, 2026 Build 42 ~5 min read

Build 42 is still receiving frequent patches. Every update carries a risk of save corruption. Here is exactly how to protect your saves so you never lose 100+ hours of progress.

Where Your Save Files Are Located

OSSave Location
WindowsC:\Users\<YourName>\Zomboid
Mac~/Zomboid
Linux~/Zomboid

Inside the Zomboid folder, your save files are in the saves subdirectory. Each save has its own folder named after the save.

How to Backup Before Updating

  1. Exit the game completely ?Do not backup while the game is running.
  2. Go to your Zomboid folder (see location table above).
  3. Copy the entire saves folder to a safe location (external drive, cloud storage, or a separate folder).
  4. Also copy the options.ini and SandboxVars.lua files if you have custom settings.
  5. Update the game through Steam.
  6. Launch the game and test your save before deleting the backup.

Never skip this step: Build 42 patches can corrupt save files. Always backup first, even for small patches.

How to Restore a Corrupted Save

  1. Exit the game.
  2. Delete the corrupted save folder in Zomboid/saves/.
  3. Paste your backup into the saves folder.
  4. Launch the game and load the restored save.

Automated Backup Strategy

  • Before every game session: Quick copy your saves folder.
  • Before every game update: Full backup to cloud storage.
  • Weekly: Keep at least 3 generations of backups (Week 1, Week 2, Week 3).
  • Use backup tools: Tools like FreeFileSync or Windows Backup can automate this process.

Pro Tip: Name your backups with dates: PZ_Backup_2026-05-09. This way you can always find a known-good save from a specific date.
