How to Fully Disable Zombie Respawn in CDDA Mode (Build 42)

Updated May 2026Build 42.17+~5 min read

CDDA (Cataclysm: Dark Days Ahead) mode is one of the most popular hardcore presets in Project Zomboid, known for its brutal survival mechanics and unforgiving zombie population. For many players, the constant zombie respawn makes long-term survival nearly impossible. Below is the complete step-by-step guide to fully disable zombie respawn in CDDA mode.

Prerequisites: This guide works for both single-player and private multiplayer servers. Public servers often lock sandbox settings, so you cannot edit respawn values there.

Step 1: Disable Respawn for a New Game

This is the most reliable method with zero bugs or compatibility issues.

  1. Launch Project Zomboid and select Solo from the main menu.
  2. Choose your spawn location (Muldraugh, Riverside, Rosewood, or West Point).
  3. Under the game mode menu, select CDDA Preset to load the hardcore settings.
  4. Click Customize Sandbox Settings to open the full settings menu.
  5. In the left-hand tab menu, scroll down and select Advanced Zombie Options (the final tab in the Zombies section).
  6. Edit the following values exactly as shown:
    • Respawn Hours: Set to 0 – Fully disables zombie respawn in all visited areas
    • Respawn Unseen Hours: Set to 0 – Disables respawn in unvisited map cells
    • Respawn Multiplier: Set to 0.0 – Eliminates any leftover respawn chance
    • Redistribute Hours: Set to 0 – Stops zombie migration between cells
  7. Click Save Settings at the bottom of the menu.
  8. Complete character creation and spawn in. All zombies you kill will stay dead permanently.

Step 2: Disable Respawn in an Existing Save

If you are already mid-run and do not want to start over:

  1. Exit your game and open the Steam Workshop.
  2. Subscribe to the Change Sandbox Options mod (well-supported mod for editing save settings).
  3. Launch Project Zomboid and enable the mod in your save file.
  4. Load into your existing game.
  5. Open the pause menu and select Edit Sandbox Settings.
  6. Navigate to Advanced Zombie Options and edit the values as listed above.
  7. Save and restart your game. Changes take effect immediately.

Step 3: Disable Respawn on a Private Server

For dedicated private servers:

  1. Shut down your server completely. Do not edit settings while running.
  2. Navigate to your server's Zomboid/Server folder.
  3. Find the file named <YourServerName>_SandboxVars.lua
  4. Open it in a text editor and edit: 
    RespawnHours = 0,
RespawnUnseenHours = 0,
RespawnMultiplier = 0.0,
RedistributeHours = 0,
  5. Save the file and restart your server.

Balanced Alternative Settings

If you want to keep some tension without constant respawns, use these popular community settings:

SettingRecommended ValueEffect
Respawn Hours1681 in-game week between respawns
Respawn Unseen Hours723 in-game days for unvisited cells
Respawn Multiplier0.05Only 5% of killed zombies respawn
Redistribute Hours48Zombies migrate every 2 days

Why this works: Cleared areas stay safe for a full week, while still making the world feel alive and dangerous long-term. This is the setup recommended for most players.

Troubleshooting

  • Zombies still respawning after setting to 0: Make sure you edited Advanced Zombie Options, not basic Zombie Population. Update to Build 42.17+ if on an unstable patch.
  • Settings won't save in existing save: Ensure the Change Sandbox Options mod is enabled and up to date. Restart your game after enabling.
  • Server settings won't apply: Server must be fully shut down before editing SandboxVars.lua. Check syntax – all lines must end with a comma.
